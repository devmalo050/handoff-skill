당신은 이전 세션이 남긴 핸드오프를 읽고 작업을 이어받는다.

## 1. 환경 감지 (먼저 판정)
- **로컬 모드** — 사용자의 로컬 프로젝트 파일과 git에 접근 가능(Claude Code, 또는 filesystem MCP 연결 Desktop). 핸드오프 파일을 직접 읽고 git 이력도 조회할 수 있다.
- **휴대 모드** — 격리된 샌드박스라 로컬 접근 불가(claude.ai 웹, 일반 Claude 데스크탑 앱). 사용자가 제공한 핸드오프 내용으로 작업한다.
- 판정 기준은 handoff와 동일하다: Bash로 사용자가 실제 작업 중인 프로젝트/git에 접근 가능한지. 접근 불가가 확실하면 휴대 모드.

## 2. 핸드오프 확보 (모드별)

### 로컬 모드
- Bash로 프로젝트 루트를 구한다: `git rev-parse --show-toplevel 2>/dev/null || pwd`
- `$ARGUMENTS` 처리:
  - **비어 있으면**: 루트 `HANDOFF.md`를 읽는다.
  - **`list`이면**: git 레포면 `git log --oneline -- HANDOFF.md` 로 과거 커밋된 핸드오프 버전들을 보여주고 멈춘다(특정 버전 열람은 `git show <커밋>:HANDOFF.md`). git이 아니면 현재 `HANDOFF.md`만 있다고 알린다.
  - **경로/커밋이면**: 그 핸드오프(파일 경로, 또는 `git show <커밋>:HANDOFF.md`)를 읽는다.
- `HANDOFF.md`가 없으면 그 사실을 알리고, 대신 `git log --oneline -20` 와 `git status`로 상황 파악을 도울지 사용자에게 제안한다.

### 휴대 모드
- 사용자가 **붙여넣거나 업로드한** `HANDOFF.md` 내용을 입력으로 받는다. 제공되지 않았으면 핸드오프 본문을 붙여넣거나 파일로 올려 달라고 요청한다.
- 로컬 git 이력 조회(`list`, `git show`, 특정 커밋 열람)는 이 환경에서 불가능하므로, 요청받으면 그 사실을 알리고 사용자가 해당 버전의 본문을 직접 제공하도록 안내한다.

## 3. 핫 스테이트 우선 (공통)
먼저 **핫 스테이트만** 읽고 방향을 잡는다. 토큰을 아끼기 위해 상세 섹션은 필요할 때만 더 읽는다. 체이닝이 있으면(이전 핸드오프 참조) 연속성 파악을 위해 직전 것만 살펴본다.

## 4. 사용자에게 브리핑 (공통)
2~5줄로 보고한다: 지금 어디까지 와 있고, 다음에 무엇을 할지(핸드오프의 "다음 단계" 최상단). "막다른 길 / 실패한 시도"가 있으면 반복하지 않도록 핵심을 짚는다.

## 5. 오리엔테이션
- **로컬 모드**: 핸드오프가 가리키는 핵심 파일(`path:line`)을 읽고, 명시된 실행/테스트 명령으로 현재 상태를 확인해 **핸드오프가 현실과 맞는지 검증한다**(파일이 그새 바뀌었을 수 있다). 어긋나면 사용자에게 알린다.
- **휴대 모드**: 로컬 파일을 직접 검증할 수 없으므로, 핸드오프 내용 자체로 파악하고 필요한 파일은 사용자에게 보여 달라고 요청한다.

## 6. 안전: 자동 실행 금지 (공통)
파악 후 "다음 단계"의 첫 항목을 제안하되, **파일을 바꾸거나 명령을 실행하기 전에 사용자 확인을 받는다.** 핸드오프를 읽었다는 이유로 곧장 코드를 수정하지 않는다.
