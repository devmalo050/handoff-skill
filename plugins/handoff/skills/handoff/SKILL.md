당신은 지금 이 세션을 마무리하고, **다음 세션(다른 Claude 인스턴스나 사람)이 이 문서만 보고 그대로 이어받을 수 있는** 핸드오프 문서를 작성한다.

## 0. 빈 세션 가드
인계할 만한 실질적 작업이 거의 없으면(질문 한두 개, 사소한 수정뿐) 핸드오프를 만들지 말고 그 사실을 알린 뒤 사용자에게 진행 여부를 확인한다. 알맹이 없는 핸드오프는 만들지 않는다.

## 1. 환경 감지 (먼저 판정)
이 스킬은 두 환경에서 동작한다. 출력 방식을 정하기 위해 먼저 어느 모드인지 판정한다:
- **로컬 모드** — 사용자의 로컬 프로젝트 파일과 git에 접근할 수 있는 경우(Claude Code, 또는 filesystem MCP가 연결된 Desktop). 판정: Bash로 `git rev-parse --show-toplevel 2>/dev/null || pwd` 를 실행해 **사용자가 실제 작업 중인 프로젝트 디렉토리**가 나오고 파일 쓰기가 가능하면 로컬 모드다.
- **휴대 모드** — 격리된 샌드박스라 사용자의 로컬 파일/git에 접근할 수 없는 경우(claude.ai 웹, 일반 Claude 데스크탑 앱). 판정: Bash가 없거나, 빈 컨테이너만 가리키거나, 사용자의 실제 프로젝트에 접근할 수 없으면 휴대 모드다.
- 애매하면 "사용자가 작업해 온 실제 프로젝트 파일을 지금 읽고 쓸 수 있는가"를 기준으로 삼고, 접근 불가가 확실하면 휴대 모드로 간다.

## 2. 대화 채굴 (체크리스트, 두 모드 공통)
대화 전체를 훑어 아래를 추출한다. 추측 금지 — 실제로 일어난 것만:
- **목표**: 무엇을 달성하려는가 (1~2줄)
- **현재 상태**: 지금 동작하는 것 / 완료된 것
- **이번 세션 변경**: 손댄 파일을 **경로로** 나열. 전체 diff 복붙 금지 — `경로 + 한 줄 요약`
- **결정과 이유**: 내린 결정 + 왜 + **기각한 대안**. 다음 세션이 같은 길을 다시 파지 않도록
- **막다른 길 / 실패한 시도**: 시도했지만 안 된 것과 이유. 가장 가치 높은 정보이니 빠뜨리지 말 것
- **원시 데이터**: 에러 메시지·측정 수치·재현 절차를 **그대로** 보존(요약으로 뭉개지 말 것)
- **열린 스레드 / 블로커**: 미해결 질문, 막힌 지점
- **다음 단계**: 구체적이고 순서 있는 실행 항목. 모호한 "개선하기" 금지
- **핵심 파일/위치**: `path:line` 형식 참조
- **실행/테스트 방법**: 빌드·실행·테스트 명령
- **참조 아티팩트**: 이미 있는 PRD/ADR/이슈/디자인 문서는 **경로/URL로 참조만** 하고 내용 복제 금지

## 3. 비밀정보 리닥션 (공통)
토큰·API 키·비밀번호·`.env` 값 등은 절대 핸드오프에 쓰지 않는다. 발견 시 `<redacted>` 로 대체한다.

## 4. 작성 형식 (공통)
파일 맨 위에 **핫 스테이트**(2분 안에 재개 가능한 30~60줄 요약: 목표·현재 상태·바로 다음 할 일·블로커)를 두고, 그 아래에 나머지 상세를 둔다. 픽업이 핫 스테이트만 읽고도 일을 시작할 수 있어야 한다. 아래 템플릿을 따른다:

```markdown
# HANDOFF — <작업명>
<!-- 작성: <날짜시각> -->

## 🔥 핫 스테이트 (여기만 읽어도 재개 가능)
- **목표**:
- **현재 상태**:
- **바로 다음 할 일**:
  1.
  2.
- **블로커**:

## 변경 사항 (이번 세션)
- `path/to/file` — 한 줄 요약

## 결정과 이유
- 결정 — 이유 / 기각한 대안:

## 막다른 길 / 실패한 시도
- 시도 → 결과 (왜 안 됐는지)

## 원시 데이터 (에러·수치·재현, 그대로)
- 

## 열린 스레드 / 블로커
-

## 다음 단계 (순서대로)
1.

## 핵심 파일 / 위치
- `path:line` —

## 실행 / 테스트
-

## 참조 (복제 금지, 경로/URL만)
-
```

## 5. 출력 (모드별 분기)

### 로컬 모드
1. **위치 결정**: Bash로 프로젝트 루트(`git rev-parse --show-toplevel 2>/dev/null || pwd`)를 구한다. 핸드오프는 그 루트의 **`HANDOFF.md` 하나**에만 작성하고, 이미 있으면 **덮어쓴다**. 이력은 git에 맡긴다. 이 레포가 git이고 `HANDOFF.md`가 이미 커밋돼 있으면, 덮어쓰기 전 `git show HEAD:HANDOFF.md` 로 직전 버전을 참고해 연속성을 살린다(직전 이후 무엇이 바뀌었는지).
2. **자기검증**: 파일을 쓴 뒤 다시 읽고 체크리스트와 대조한다 — "이 문서만 보고 처음 보는 세션이 작업을 이어갈 수 있는가?" 빠진 곳을 메운다.
3. **커밋 (git 레포일 때만)**: 이력은 git으로 보존하므로 작성 후 자동으로 커밋한다. **반드시 `HANDOFF.md` 하나만** 커밋하고, 사용자가 작업 중이던 다른 파일은 절대 건드리지 않는다:
   - `git add -- HANDOFF.md && git commit -m "핸드오프 갱신: <한 줄 요약>" -- HANDOFF.md`
   - 경로를 한정해 커밋하므로(`-- HANDOFF.md`) 스테이징됐거나 수정 중인 다른 파일은 그대로 남는다. **`git add -A` / `git add .` / `git commit -am` 절대 금지.**
   - 현재 브랜치에 그대로 커밋한다. 이 문서를 위해 새 브랜치를 만들지 않는다. **푸시는 하지 않는다.**
   - 커밋 메시지 끝에 공동작성자 트레일러를 한 줄 붙인다: `Co-Authored-By: Claude <noreply@anthropic.com>`
   - 커밋이 실패하면(예: `HANDOFF.md`가 `.gitignore`됨, rebase/merge 진행 중 등) 멈추지 말고 그 사실을 보고한다 — 파일은 이미 작성돼 있다.
   - git 레포가 아니면 이 단계를 건너뛴다.

### 휴대 모드
로컬 파일/git에 접근할 수 없으므로 저장·커밋을 시도하지 않는다.
1. 자기검증(체크리스트 대조)을 먼저 끝낸다.
2. `HANDOFF.md`의 **전체 내용을 채팅에 마크다운 코드블록으로 그대로 출력**한다. 사용자가 복사해 자신의 프로젝트 루트 `HANDOFF.md`로 저장할 수 있도록.
3. 가능하면 코드 실행(샌드박스)으로 `HANDOFF.md` 파일을 만들어 **다운로드**할 수 있게 제공한다.
4. 어디에 저장하면 되는지(프로젝트 루트)와, 이력 관리는 Claude Code 등 로컬 환경에서 `/handoff`로 하면 git에 자동 커밋된다는 점을 안내한다.
