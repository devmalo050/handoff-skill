argument-hint: "[특정 핸드오프 경로 | list]"

당신은 이전 세션이 남긴 핸드오프를 읽고 작업을 이어받는다.

## 1. 핸드오프 찾기
- Bash로 프로젝트 루트를 구한다: `git rev-parse --show-toplevel 2>/dev/null || pwd`
- `$ARGUMENTS` 처리:
  - **비어 있으면**: 루트 `HANDOFF.md`를 읽는다.
  - **`list`이면**: git 레포면 `git log --oneline -- HANDOFF.md` 로 과거 커밋된 핸드오프 버전들을 보여주고 멈춘다(특정 버전 열람은 `git show <커밋>:HANDOFF.md`). git이 아니면 현재 `HANDOFF.md`만 있다고 알린다.
  - **경로/커밋이면**: 그 핸드오프(파일 경로, 또는 `git show <커밋>:HANDOFF.md`)를 읽는다.
- `HANDOFF.md`가 없으면 그 사실을 알리고, 대신 `git log --oneline -20` 와 `git status`로 상황 파악을 도울지 사용자에게 제안한다.

## 2. 핫 스테이트 우선
먼저 **핫 스테이트만** 읽고 방향을 잡는다. 토큰을 아끼기 위해 상세 섹션은 필요할 때만 더 읽는다. 체이닝이 있으면(이전 핸드오프 참조) 연속성 파악을 위해 직전 것만 살펴본다.

## 3. 사용자에게 브리핑
2~5줄로 보고한다: 지금 어디까지 와 있고, 다음에 무엇을 할지(핸드오프의 "다음 단계" 최상단). "막다른 길 / 실패한 시도"가 있으면 반복하지 않도록 핵심을 짚는다.

## 4. 오리엔테이션
핸드오프가 가리키는 핵심 파일(`path:line`)을 읽고, 명시된 실행/테스트 명령으로 현재 상태를 확인해 **핸드오프가 현실과 맞는지 검증한다**(파일이 그새 바뀌었을 수 있다). 어긋나면 사용자에게 알린다.

## 5. 안전: 자동 실행 금지
파악 후 "다음 단계"의 첫 항목을 제안하되, **파일을 바꾸거나 명령을 실행하기 전에 사용자 확인을 받는다.** 핸드오프를 읽었다는 이유로 곧장 코드를 수정하지 않는다.
